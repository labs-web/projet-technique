# Workflow de travail : Les 3 Grands Temps du Projet



## Définitions et Concepts Clés

### 1. Skill (Compétence)
C'est le **"Savoir-Faire"**. Un Skill est un ensemble de directives, de connaissances et d'outils permettant à l'agent d'agir en tant qu'expert dans un domaine précis (ex: UX Design, Backend Laravel, Intégration CSS). Structurellement, c'est un dossier contenant un fichier `SKILL.md`.

### 2. Workflow (Processus)
C'est le **"Savoir-Agir"**. Un Workflow est une séquence d'étapes logiques ordonnancées pour atteindre un objectif de production (ex: Créer une page, Déployer l'app).

### La Contrainte "Skill-Driven"
> **Règle Absolue** : Chaque étape d'un workflow DOIT être pilotée par un Skill identifié.
L'agent ne "bidouille" pas ; il endosse un rôle d'expert pour chaque micro-tâche pour garantir l'excellence à chaque étape.

## Vue d'Ensemble du Processus

Notre méthode de travail est séquentielle et rigoureuse. Elle sépare nettement la conception, l'intégration graphique et le développement logique pour éviter la dette technique.

## Workflow

Nous découpons le travail en 3 "Meta-Workflows". Chaque étape est pilotée par un **Skill (Expert)** spécifique.

---

### 1. Initialisation (`/initialisation`)
*Ce workflow ne s'exécute qu'une seule fois.*

**1. Infra**
*   **Action** : Installation de Laravel, Git, Base de données.
*   **Skill** : `Architecte Technique`

**2. Structure**
*   **Action** : Création des dossiers clés (`Services`, `Partials`) et config des Outils.
*   **Skill** : `Architecte Technique`

**3. Config Frontend**
*   **Action** : Installation de Tailwind, Vite et structure des assets.
*   **Skill** : `Expert Tailwind`

---

### 2. Maquettage UI (`/maquettage`)
*Condition : À lancer si les interfaces (UI) n'existent pas encore pour la fonctionnalité visée.*

**1. Conception**
*   **Action** : Définir le besoin, rédiger la User Story, dessiner le Wireframe.
*   **Skill** : `Concepteur UI`

**2. Inventaire**
*   **Action** : Vérifier si les composants (Boutons, Cards...) existent dans le `ui-kit`.
*   **Skill** : `Concepteur UI`

**3. Création**
*   **Action** : Coder les composants manquants en HTML/CSS statique (Pixel Perfect).
*   **Skill** : `Créateur UI`

---

### 3. Développement (`/developpement`)
*Condition Préalable : Les UI doivent exister (sinon -> voir Maquettage).*

Ce workflow suit strictement le flux de données (du bas vers le haut) :

**1. Data**
*   **Action** : Trouver/Créer le bon chemin d'accès aux données (Requête SQL, Eloquent).
*   **Skill** : `Expert Données (Model)`

**2. Logique**
*   **Action** : Encapsuler la règle métier dans une méthode dédiée.
*   **Skill** : `Expert Service`

**3. Orchestration**
*   **Action** : Créer la Route et le Contrôleur qui appelle le Service.
*   **Skill** : `Développeur Laravel`

**4. Rendu**
*   **Action** : Connecter les données aux composants UI existants (Atom/Molecule).
*   **Skill** : `Intégrateur Blade`

> **Note** : Il est interdit de créer du HTML/CSS dans l'étape 4. On ne fait qu'assembler des briques existantes.

---

## Les Compétences (Skills) par Couche

Pour garantir la qualité, nous divisons l'expertise technique par couche de responsabilité.

### Pôle Architecture & Design
*   **Architecte Technique** : Garant de l'infrastructure et de l'initialisation.
*   **Concepteur UI** : Garant de l'expérience utilisateur (UX) et du plan.
*   **Créateur UI** : Garant du style (CSS/Tailwind) et de l'accessibilité.

### Pôle Développement (Backend & Frontend)
*   **Expert Données** : Spécialiste Eloquent/SQL. Il s'assure que les données sont intègres et optimisées.
*   **Expert Service** : Spécialiste de la "Business Logic". Il écrit le code pur, sans http, réutilisable.
*   **Développeur Laravel** : Le chef d'orchestre (Routes/Controllers). Il fait le lien HTTP.
*   **Intégrateur Blade** : Spécialiste du templating. Il assemble les **Partials** et **Layouts** sans casser le design.
